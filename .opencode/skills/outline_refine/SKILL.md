# 大纲优化Skill

## 功能
分析大纲的完整性和合理性，提供完善建议和引导性问题。

## 使用场景
- 收到用户的初始大纲后
- 大纲需要进一步细化时
- 发现大纲逻辑问题需要修复时

## 分析维度

### 1. 完整性检查
检查以下要素是否存在：
- [ ] 明确的故事类型
- [ ] 清晰的核心冲突
- [ ] 主角和关键配角
- [ ] 故事背景设定
- [ ] 明确的结局方向
- [ ] 主要情节线

### 2. 合理性检查
- 情节发展是否符合逻辑
- 角色动机是否合理
- 时间线是否连贯
- 冲突是否有足够的强度
- 结局是否水到渠成

### 3. 节奏评估
- 章节数量是否合适（10-30章为常见范围）
- 高潮位置是否合理（通常在75%-85%）
- 是否有适当的铺垫和缓冲
- 每章是否有明确目标

## 输入
```json
{
  "大纲": "当前的大纲内容",
  "用户偏好": "用户的特别要求（如有）"
}
```

## 输出
```json
{
  "完整性评分": "0-100分",
  "缺失要素": ["缺失的要素列表"],
  "合理性问题": ["逻辑问题列表"],
  "节奏评估": "节奏是否合适",
  "引导性问题": [
    {
      "维度": "背景设定",
      "问题": "故事发生在什么时代？有特殊的世界观吗？",
      "重要性": "高"
    }
  ],
  "改进建议": [
    "建议1",
    "建议2"
  ]
}
```

## 使用方法
1. 接收大纲内容
2. 按维度进行检查
3. 生成评分和问题列表
4. 按重要性排序引导问题（高优先级先问）
5. 提供改进建议

## 注意事项
- 不要一次性生成太多问题（3-5个为宜）
- 问题要具体，不能太宽泛
- 优先询问核心要素（冲突、角色、背景）
- 保持建议的可操作性
