# 章节创作Skill

## 功能
融合大纲目标、角色卡和文风指南，创作符合要求的章节内容。

## 使用场景
- 编写新章节时
- 重写现有章节时
- 根据反馈修改章节时

## 创作前检查清单

### 信息确认
- [ ] 章节目标已明确
- [ ] 相关角色卡已读取
- [ ] 文风指南已加载
- [ ] 上下文（前文概要）已了解
- [ ] 用户的特殊要求已知晓

### 规划要点
- [ ] 开头如何承接或开场
- [ ] 核心冲突是什么
- [ ] 哪些角色出场、做什么
- [ ] 何时达到章节高潮
- [ ] 结尾如何为下章铺垫

## 创作流程

### 第一步：确定章节结构
```
开头（100-300字）
├─ 如果是第一章：介绍场景/主角/引发事件
└─ 如果不是第一章：承接上文，说明时间地点

发展（篇幅最大）
├─ 引入冲突或任务
├─ 角色展开行动
├─ 小挫折或小胜利
├─ 推进剧情

高潮（章节关键点）
├─ 最大冲突或转折
├─ 角色面临抉择
├─ 揭露重要信息

结尾（100-200字）
├─ 确认本章成果
└─ 为下章留线索/悬念
```

### 第二步：填充内容
按照文风指南的规范：
- 使用符合文风的句式和词汇
- 控制段落长度
- 保持对话/描写/叙述的比例
- 运用推荐的表达方式

### 第三步：角色一致性
- 检查每个角色的言行是否符合角色卡
- 角色之间的互动符合关系设定
- 角色动机推动情节发展

### 第四步：自检优化
- 字数是否合适（建议2000-5000字/章）
- 是否达成章节目标
- 与大纲是否一致
- 文风是否正确
- 是否有前后矛盾

## 输入
```json
{
  "章节目标": "本章要达成什么",
  "关键角色": ["角色1", "角色2"],
  "主要事件": "发生的事情",
  "文风指南": "完整的文风指南",
  "角色卡": {"角色1": "信息", "角色2": "信息"},
  "上下文": {
    "前一章概要": "...",
    "时间线": "当前时间点",
    "地点": "当前地点"
  },
  "用户要求": "特殊要求（如有）"
}
```

## 输出
```json
{
  "章节标题": "标题",
  "正文": "完整章节文本",
  "创作说明": {
    "达成目标": "如何达成",
    "关键情节": "3-5个关键情节点",
    "角色发展": "角色的变化",
    "伏笔": "埋下的伏笔（如有）"
  },
  "字数": "实际字数"
}
```

## 质量标准

### 内容质量
- 章节目标必须达成
- 情节推进合理
- 冲突和转折有意义
- 对话自然流畅

### 文风质量
- 严格遵循文风指南
- 语言表达准确
- 句式符合规范
- 用词得当

### 连贯性
- 与前后章节衔接自然
- 角色行为一致
- 时间地点准确
- 无逻辑矛盾

## 修改流程
1. 理解用户反馈
2. 定位需要修改的部分
3. 重新创作修改内容
4. 确保修改不影响其他部分
5. 整合后输出新版本

## 注意事项
- 不要偏离章节目标
- 角色不能OOC（Out of Character）
- 遵循文风指南的所有规范
- 确保前后章节连贯
- 支持用户修改任何部分
- 创作完成后要自检
