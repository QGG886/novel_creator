# 章节创作Skill

## 功能
融合大纲目标、角色卡和文风指南，创作符合要求的章节内容。

## 使用场景
- 编写新章节时
- 重写现有章节时
- 根据反馈修改章节时

## 创作前检查清单

### 信息确认
- [ ] 章节目标已明确
- [ ] 相关角色卡已读取
- [ ] 文风指南已加载
- [ ] 上下文（前文概要）已了解
- [ ] 用户的特殊要求已知晓
- [ ] **前两章剧情分析已完成（从第3章开始）**
- [ ] 创作指导已获得（避免重复和矛盾）

### 规划要点
- [ ] 开头如何承接或开场
- [ ] 核心冲突是什么
- [ ] 哪些角色出场、做什么
- [ ] 何时达到章节高潮
- [ ] 结尾如何为下章铺垫
- [ ] **需要避免的内容（根据创作指导）**
- [ ] **需要保持一致的关键点（根据创作指导）**
- [ ] **需要推进的情节（根据创作指导）**

## 创作流程

### 第零步：前两章剧情分析（从第3章开始）
```
调用chapter_context_analysis Skill
├─ 读取前两章内容
├─ 提取关键信息（时间线、地点、角色状态、情节进展、伏笔）
├─ 生成创作指导
│   ├─ 开头应承接的方式
│   ├─ 核心冲突设计建议
│   ├─ 需要避免的内容（重复分析、重复介绍、矛盾点）
│   ├─ 需要保持一致的关键点
│   └─ 需要推进的情节
└─ 识别潜在风险
```

### 第一步：确定章节结构
```
开头（100-300字）
├─ 如果是第一章：介绍场景/主角/引发事件
└─ 如果不是第一章：承接上文，说明时间地点

发展（篇幅最大）
├─ 引入冲突或任务
├─ 角色展开行动
├─ 小挫折或小胜利
└─ 推进剧情

高潮（章节关键点）
├─ 最大冲突或转折
├─ 角色面临抉择
└─ 揭露重要信息

结尾（100-200字）
├─ 确认本章成果
└─ 为下章留线索/悬念

### 第二步：填充内容
按照文风指南的规范：
- 使用符合文风的句式和词汇
- 控制段落长度
- 保持对话/描写/叙述的比例
- 运用推荐的表达方式

### 第三步：角色一致性
- 检查每个角色的言行是否符合角色卡
- 角色之间的互动符合关系设定
- 角色动机推动情节发展

### 第四步：自检优化
- 字数是否合适（建议2000-5000字/章）
- 是否达成章节目标
- 与大纲是否一致
- 文风是否正确
- 是否有前后矛盾

## 质量标准

### 内容质量
- 章节目标必须达成
- 情节推进合理
- 冲突和转折有意义
- 对话自然流畅

### 文风质量
- 严格遵循文风指南
- 语言表达准确
- 句式符合规范
- 用词得当

### 连贯性
- 与前后章节衔接自然
- 角色行为一致
- 时间地点准确
- 无逻辑矛盾

## 修改流程
1. 理解用户反馈
2. 定位需要修改的部分
3. 重新创作修改内容
4. 确保修改不影响其他部分
5. 整合后输出新版本

## 注意事项
- 不要偏离章节目标
- 角色不能OOC（Out of Character）
- 遵循文风指南的所有规范
- 确保前后章节连贯
- 支持用户修改任何部分
- 创作完成后要自检
- **从第3章开始，创作前必须先调用chapter_context_analysis Skill分析前两章**
- **必须根据创作指导避免重复前文已分析或介绍的内容**
- **保持时间线、地点、角色状态的一致性**
- **注意回收前文埋下的伏笔**
