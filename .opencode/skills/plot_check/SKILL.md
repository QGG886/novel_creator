# 情节漏洞检查Skill

## 功能

检查章节中的情节是否完整，伏笔是否回收，关键信息是否解释，发现问题并提供具体的修改方案。

---

## 检查维度

### 1. 未解决的伏笔
- 前文埋下的伏笔是否回收

### 2. 未解释的关键信息
- 关键信息是否有解释

### 3. 突然出现的解法
- 问题的解决是否合理

### 4. 缺少铺垫的转折
- 转折是否有足够铺垫

---

## 输入格式

```json
{
  "章节内容": "章节完整文本",
  "章节信息": {
    "章节号": 4,
    "章节大纲": "第4章的大纲内容"
  },
  "已写章节": ["章节1", "章节2", "章节3"],
  "大纲": "完整大纲"
}
```

---

## 修改方案生成规则

### 未解决的伏笔

**问题示例**：前文埋下伏笔但未回收

**修改方案**：
- 方案类型：增加
- 具体操作：增加伏笔回收
- 修改后内容：
  - 在合适的地方回收伏笔
- 操作说明：
  1. 识别未回收的伏笔
  2. 设计合理的回收方式
  3. 增加相关内容

### 突然出现的解法

**问题示例**：问题突然解决，缺乏铺垫

**修改方案**：
- 方案类型：增加
- 具体操作：增加解法的铺垫
- 修改后内容：
  - 在前文中增加线索和暗示
- 操作说明：
  1. 分析解法是什么
  2. 设计合理的铺垫方式
  3. 增加相关内容

---

## 严重程度分级

### A类（严重）
- 重要伏笔未回收
- 关键信息未解释
- 核心情节解法突然

### B类（中等）
- 次要伏笔未回收
- 一般信息未解释

### C类（轻微）
- 细节伏笔可以更完善
