# 逻辑漏洞检查Skill

## 功能

检查章节中的因果关系、行为动机、事件发展是否符合逻辑，发现问题并提供具体的修改方案。

---

## 检查维度

### 1. 因果关系
- 事件之间是否有合理的因果关系
- 原因和结果是否匹配

### 2. 行为动机
- 角色行为是否有充分动机
- 动机是否合理

### 3. 事件发展
- 事件发展是否有依据
- 是否有逻辑跳跃

### 4. 推理过程
- 角色的推理过程是否合理
- 结论是否有充分依据

## 修改方案生成规则

### 因果关系不成立

**问题示例**：结果A出现，但缺乏足够的原因B

**修改方案**：
- 方案类型：增加
- 具体操作：增加原因或增强原因强度
- 修改后内容：
  - 在结果出现前增加充分的原因
- 操作说明：
  1. 分析结果需要什么原因
  2. 增加原因情节
  3. 确保原因和结果匹配

### 行为动机不合理

**问题示例**：角色做出某个行为，但动机不足

**修改方案**：
- 方案类型：增加
- 具体操作：增加角色动机
- 修改后内容：
  - 增加角色动机相关的情节
  - 增加心理活动
- 操作说明：
  1. 分析行为需要什么动机
  2. 增加动机相关内容
  3. 确保动机充分

## 严重程度分级

### A类（严重）
- 核心情节逻辑漏洞
- 主要角色行为缺乏动机
- 逻辑链条断裂

### B类（中等）
- 次要情节逻辑问题
- 一般行为动机不足

### C类（轻微）
- 细节逻辑不够严密
