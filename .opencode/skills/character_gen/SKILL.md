# 角色卡格式化工具

## 功能
将角色设计信息格式化为标准化的角色卡文档。

## 使用场景
- Agent完成角色设计后，需要生成标准格式文档
- 批量生成角色卡时
- 统一角色卡格式时

## 输入格式
```json
{
  "角色设计": {
    "角色名": "角色名称",
    "基本信息": {
      "年龄": "",
      "性别": "",
      "职业": "",
      "外貌": ""
    },
    "性格特征": [
      {"特征": "特征1", "表现": "具体表现"},
      {"特征": "特征2", "表现": "具体表现"}
    ],
    "背景故事": "",
    "动机": {
      "内在": "",
      "外在": ""
    },
    "能力技能": [],
    "弱点缺陷": [],
    "关系网络": {},
    "角色弧光": "",
    "标志性特征": ""
  },
  "模板级别": "S/A/B/C"
}
```

## 输出格式
```json
{
  "状态": "成功",
  "角色卡": {
    "角色名": "角色名称",
    "文件路径": "characters/角色名.md"
  },
  "格式版本": "v1.0"
}
```

## 输出文档格式（S/A级）
```markdown
# 角色名

## 基本信息
- **年龄**：
- **性别**：
- **职业**：
- **外貌**：

## 性格特征
1. **特征1**：具体表现
2. **特征2**：具体表现
3. **特征3**：具体表现

## 背景故事
[详细描述]

## 动机
- **内在驱动力**：
- **外在目标**：

## 能力/技能
- 能力1
- 能力2

## 弱点/缺陷
- 缺陷1
- 缺陷2

## 关系网络
- 与角色A：[关系]
- 与角色B：[关系]

## 角色弧光
[成长轨迹描述]

## 标志性特征
[台词风格、习惯动作、口头禅]
```

## 注意事项
1. 本工具只负责格式化，不负责角色设计
2. 所有设计内容必须由Agent提供
3. 遵循COMMON.md的文件命名规范
